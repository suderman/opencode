---
description: Commit the currently staged changes
agent: main
---

Commit the currently staged changes only.

Inspect the staged diff carefully with `git diff --staged`.

Do not stage, unstage, edit, format, or modify any files.

Write a clear, detailed commit message that accurately describes what is
included in the staged diff.

Before committing:

- confirm that there are staged changes
- summarize what is staged
- choose an appropriate conventional commit type if it fits naturally

Then create exactly one commit from the staged changes.
